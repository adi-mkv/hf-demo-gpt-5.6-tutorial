Step 01 / 02

What Is HyperFrames

↗ hyperframes.heygen.com ↗ GitHub

HyperFrames is an open-source framework from HeyGen that turns HTML, CSS, and JavaScript into a deterministic MP4. A composition is just an index.html file with data attributes marking start time, duration, and track for each element. HyperFrames advances a headless browser to each frame's timestamp, screenshots it, and stitches the frames together. Animation can come from CSS, GSAP, canvas, Three.js, Lottie, or a custom adapter — whatever the composition calls for.

Because a composition is plain HTML rather than a proprietary timeline format, an AI coding agent like Claude Code or Codex can read, write, and edit it the same way it edits any other web project. That's the whole premise: the agent is already fluent in HTML, so let it work in HTML. Below is how to start a blank project:

INSTALL 
npx hyperframes init my-video --non-interactive --example blank
cd my-video
npx hyperframes preview   # opens HyperFrames Studio in the browser, hot-reloads on save

preview starts the dev server and opens the composition in HyperFrames Studio. Save a change to index.html and the preview updates instantly — no manual refresh, no build step.

Step 02 / 02

Hand It To An Agent

Once Studio is running locally, the fastest way to learn this particular video's system is to point an agent at a finished repo and have it read through the composition before changing anything. This is the prompt I use to get Claude Code or Codex to inspect a project like this one, file by file, and report back before it touches any code.

PROMPT 
Clone this repo: [REPO LINK]
Open the HyperFrames project and launch Studio:
cd gpt-5.6
npm run dev

Then inspect the main project files to understand how this video is built with HTML, CSS, GSAP, canvas, and HyperFrames:
- `gpt-5.6/index.html` - master HyperFrames composition entrypoint
- `gpt-5.6/src/scenes.js` - scene markup and on-screen copy
- `gpt-5.6/src/styles.css` - visual system, layout, typography, cards, backgrounds, logo styling
- `gpt-5.6/src/timeline.js` - GSAP timeline, camera sweeps, text animation, particle/logo morph, scene timing
- `gpt-5.6/assets/` and `gpt-5.6/fonts/` - source assets and local typefaces

Pay close attention to reusable techniques like the off-canvas camera world, vertical 9:16 layout, soft background movement, translucent cards, rolling number counter, soft blur-in text reveals, staggered typography, GSAP scene timing, and the canvas particle swarm that morphs into the final logo/CTA.

Use it as inspiration for a new HyperFrames video, not as something to copy exactly. After inspecting it, summarize the structure, the timeline logic, the most reusable techniques, and any adaptation ideas. Tell me when you're ready.
